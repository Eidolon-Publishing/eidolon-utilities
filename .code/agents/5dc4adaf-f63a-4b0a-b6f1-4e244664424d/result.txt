**Plan**

Provide a prioritized, phased implementation plan that tightens cancellation/error handling, clarifies hook policy, and validates tween semantics before running timelines so the executor becomes production-ready and maintainable.

## Requirements
- Immediate cancellation of in-flight steps via abort-aware executors and handles.
- Distinct outcomes for success, cancellation, and runtime failures.
- Hooks honor the configured `errorPolicy`; only synchronous errors should stop or continue timelines per policy.
- Sequence JSON validation must cover registered tween validators before execution.
- Backward-compatible API that still supports named timelines, socket sync, and existing consumers.
- Maintainable structure with clear file ownership.

## Scope
- In: `scripts/tween/core/timeline/` (execution engine, handle, schema), executor setup, registry, and hooks that orchestrate sockets and public API.
- Out: unrelated modules/repacks, non-tween features.

## Files and entry points
- `scripts/tween/core/timeline/TweenTimeline.js` – execution flow, hooks, error/cancel semantics.
- `scripts/tween/core/timeline/TimelineHandle.js` – handle contract, statuses, abort signal handling.
- `scripts/tween/core/timeline/schema.js` – JSON validation, compile logic, tune for validator enforcement.
- `scripts/tween/core/executors/sequence.js` – validation/execution bridge, ensures behavior matches updated schema.
- `scripts/tween/core/registry.js` – provide validator metadata for semantic checks.
- `scripts/tween/hooks.js` – socket behavior, API exposure, ensures broadcast/cancel messaging align with new contract.

## Data model / API changes
- **TimelineHandle.finished** resolves with a richer result (e.g., `{ status: "completed"|"cancelled"|"failed", reason?: Error }`) instead of boolean.
- **TweenTimeline.run** returns handle whose cancel and hooks (beforeAll/before/after) are wrapped by errorPolicy-aware guards.
- **Sequence JSON** carries optional metadata (e.g., `validationVersion` or `requires`) enabling registry validators to run before compile.
- Backward compatibility: default `finished` behavior stays boolean for existing call sites that don’t inspect (maybe via helper getter or metadata) and `errorPolicy` default remains `abort`.

## Action items
[ ] Phase 1 – **Instrumentation & cancellation contract**: define richer handle result, propagate abort signal to each tween execute call (via wrapper or controller) so entry executors can cancel mid-flight and `cancel()` resolves immediately without waiting for Promise.all.
[ ] Phase 2 – **Error semantics separation**: differentiate cancellation vs execution failure in the handle result and errorPolicy logic; make step hooks respect policy (wrap hook invocations, swallow vs throw errors based on policy) and ensure `errorPolicy` continues only affects entries/hook-level errors as documented.
[ ] Phase 3 – **Semantic validation layer**: extend registry to expose `validate` per tween, invoke before timeline execution (both direct and socket-compiled flows) and fail fast with descriptive errors without running mutated timeline.
[ ] Phase 4 – **Socket/registry alignment**: ensure socket broadcast/cancel respects new handle statuses, timeline registry cancels previous runs cleanly, and the public API documents new handle result semantics for consumers.
[ ] Phase 5 – **Documentation/tests/manual**: update inline docs, add unit/integration tests for cancellation/error outcomes, and manual Foundry checks for named timelines via UI sockets.

## Testing and validation
- **Unit**: mock tweens with abort-responsive promises to verify immediate cancel, verify `finished` result statuses, and ensure hooks behave per `errorPolicy`.
- **Integration**: run timeline sequences through `sequence` executor and socket handler, mocking registry validation failure to confirm early rejection.
- **Manual Foundry**: trigger named timeline from UI, cancel via API call and via socket cancellation, and observe broadcast behavior/logging; test failure path (invalid tween params) to ensure error remains localized and timeline cancels without hanging.
- **Regression**: ensure default sequence JSON still accepted, timeline serialization unchanged, and existing `onComplete/onCancel` flows run.

## Risks and edge cases
- Race between cancellation and step completion might still leave detached tweens running; ensure cleanup logic waits/surfaces warnings without blocking.
- Registry validation introduced at runtime must not break lazy-registered tweens; handle missing validator by deferring to existing behavior.
- Changing `finished` return structure requires backward-compatible accessor; avoid breaking third-party code.

- After implementation, release as patch after automated tests and manual sockets confirmed.
